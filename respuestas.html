  2. Hay 3 formas de integrar JS en HTML

    • En el head del HTML. Ejemplo: <script type="text/javascript">let Animal = 'Perro'</script>
    • En el body del HTML. Ejemplo: <script type="text/javascript">let Fruta = 'Cereza'</script>
    • Utilizando un archivo externo con extensión .js y enlazarlo a través de la etiqueta <script>. Generalmente se pone en el <head>, aunque también puede ir en el <body>. Ejemplo: <script src="JS/script.js></script>
  3. Para las variables se utilizan nombres que simbolicen su valor, por lo que no llevara un nombre cualquiera. JavaScript distingue entre mayúsculas y minúsculas. Para declarar una variable existen tres formas, con var, con let o con const que le dará a la variable un valor constante que no se podrá modificar luego.

    Hay 7 tipos de datos:

    • Undefined: cuando un dato no esté definido.
    • Boolean: un valor true o false (verdadero o falso).
    • Number: un número entero o decimal.
    • String: un valor de texto conformado por mas de un caracter.
    • BigInit: un número del tipo entero que tendrá precisión arbitraria.
    • Symbol: devuelve una propiedad única para un objeto.
    • Null: determina valor nulo.

    Las expresiones regulares son patrones que se utilizan para hacer coincidir combinaciones de caracteres en cadenas.

    • exec(): ejecuta una búsqueda por una coincidencia en una cadena.
    • test(): prueba una coincidencia en una cadena.
    • match(): devuelve un arreglo que contiene todas las coincidencias
    • matchAll(): Devuelve un iterador que contiene todas las coincidencias
    • search(): Prueba una coincidencia en una cadena.
    • replace(): Ejecuta una búsqueda por una coincidencia en una cadena y reemplaza la subcadena coincidente con una subcadena de reemplazo.
    • replaceAll(): Ejecuta una búsqueda de todas las coincidencias en una cadena y reemplaza las subcadenas coincidentes con una subcadena de reemplazo.
    • split(): utiliza una expresión regular o una cadena fija para dividir una cadena en un arreglo de subcadenas.

  4. Formas de declarar una funcion:

    • Function declaration: este tipo de función se creará con la palabra reservada function, seguido obligatoriamente por un nombre, que identificará a nuestra función, una lista de parámetros entre paréntesis, y el símbolo de las llaves {}. Qué será el que delimite el contenido de nuestro conjunto de sentencias.
    • Function expression: la sintaxis para crear estas funciones es similar a la anterior. La única diferencia es que la definición de nuestra nueva función no comienza por instrucción function y el nombre de la función es opcional.
    • IIFE (Immediately Invoked Function Expression): Este tipo de funciones se ejecutan inmediatamente y no son accesibles posteriormente. Para crear una función de este tipo deberemos crearla en un operador de agrupación (). Seguido de (), lo que posibilitará el interpretado directamente en el motor de JavaScript.
    • Shorthand method definition: para crear este tipo de función debemos asignar un nombre de función seguido de una lista de parámetros entre paréntesis y los símbolos de llaves para delimitar el cuerpo de las instrucciones.
    • Arrow function: la forma de crear estas funciones es la siguiente: Primero definiremos la lista de parámetros, en caso de ser necesario, entre paréntesis seguido del símbolo => y las {} para indicar las instrucciones que se van a realizar.
    • Generator function: nos permite parar o salir de la función en un punto dentro del conjunto de instrucciones que forman nuestra función y retornar posteriormente la ejecución desde el punto en el que paramos anteriormente. La sintaxis de estas funciones es igual a las "function declaration" o "function expression". Sólo debemos utilizar el símbolo * al comienzo de la definición de nuestra función.
    • Function constructor: podremos crear una variable que invoque a un objeto Function. Cuando invoquemos a este objeto podremos enviar todos los argumentos que deseemos. Los primeros n argumentos serán los parámetros de nuestra función y el último argumento será el código de nuestra función.

  5. Tipos de operadores:

    • Operadores unarios: es una operación con un solo operando.
    • Operadores aritméticos: toman valores numéricos (ya sean literales o variables) como sus operandos y devuelven un solo valor numérico.
    • Operadores relacionales: compara sus operandos y devuelve un valor 'boolean' basado en si la comparación es verdadera o no.
    • Operadores de igualdad: el resultado de evaluar un operador de igualdad siempre es de tipo Boolean basado en si la comparación es verdadera.
    • Operadores de desplazamiento de bits: sirve para cambiar todos los bits del operando.
    • Operadores binarios bit a bit: tratan a sus operandos como un conjunto de 32 bits (ceros y unos) y devuelven valores numéricos estándar de JavaScript.
    • Operadores lógicos binarios: se utilizan normalmente con valores booleanos (lógicos); cuando lo son, devuelven un valor booleano.
    • Operador condicional (ternario): devuelve uno de dos valores según el valor lógico de la condición.
    • Operadores de asignación: asigna un valor a su operando izquierdo basándose en el valor de su operando derecho.
    • Operador coma: permite evaluar múltiples expresiones en una sola declaración y devuelve el resultado de la última expresión.

    La conversión entre tipos ocurre cuando los operadores y funciones automáticamente son convertidos al tipo correcto de valor. Estos tipos son 'ToString' para pasar cualquier dato a representarlo de forma de texto. 'ToNumber' para pasar cualquier dato en funciones matemáticas y expresiones a una representación numérica. 'ToBoolean' pasar datos en operaciones lógicas a una representación con valor de verdad o falso.

  6. Una estructura de control es una herramienta que tienen los lenguajes de programación y que se utiliza para controlar el flujo de las operaciones durante el programa. Ejemplos:

    • if: la sentencia if especifica un bloque de código que se ejecuta si una condición es cierta y da la oportunidad de ejecutar otro bloque de código distinto si resulta que no es cierta.
    • else: la parte de else es opcional, y especifica un bloque de código que se ejecuta si la condición es falsa. Siempre acompañado de un 'IF'
    • Operador ternario(?): es una alternativa de condicional if/else de una forma mucho más corta y, en muchos casos, más legible.
    • switch: Se utiliza para hacer diferentes acciones basadas en diferentes condiciones. Consiste en darle un valor y comprobar si ese valor coincide con algún caso. En el caso que coincida, ejecutamos un bloque de código específico.

  7. El modelo de objeto de documento (DOM) es una interfaz de programación para HTML. Facilita una representación estructurada del documento y define de qué manera los programas pueden acceder, al fin de modificar, tanto su estructura, estilo y contenido. El DOM da una representación del documento como un grupo de nodos y objetos estructurados que tienen propiedades y métodos. Esencialmente, conecta las páginas web a scripts o lenguajes de programación.

    El dom se creará una vez que el navegador web lea todos los elementos HTML y los regrese como un objeto, y con esto JS podrá mostrar una representación de todo lo que tenga el archivo de hipermarcado de texto.

  8. El Object Map en JS tiene la función de ir guardando un valor como clave seguido de otro, que será la función de dicha clave. Es un tipo de array donde se formará un tipo de recuadro con filas y columnas para almacenar datos, y que después se podrá trabajar con cada función de su respectiva clave.
  9. Todos los eventos que trabaja JavaScript son:
    • Abort (onabort): evento que se produce cuando se detiene la carga de algo en la página.
    • Blur (onblur): evento que se produce cuando un elemento pierde el foco de la aplicación.
    • Change (onchange): evento que se produce cuando cambia el estado de un elemento de un formulario
    • Click (onclick): evento que se produce cuando se pulsa o se hace click en un botón o un enlace.
    • Dragdrop (ondragdrop): evento que se produce cuando un usuario suelta algo que había arrastrado sobre la página web.
    • Error (onerror): evento que se produce cuando no se puede cargar un documento o una imagen y esta queda rota.
    • Focus (onfocus): evento que se produce cuando un elemento de la página o la ventana ganan el foco de la aplicación.
    • Keydown (onkeydown): evento que se produce cuando un usuario presiona una tecla, independientemente que la suelte o no. Se produce en el momento de la pulsación.
    • Keypress (onkeypress): evento que se produce cuando el usuario deja pulsada una tecla un tiempo determinado.
    • Keyup (onkeyup): evento que se produce cuando el usuario deja de apretar una tecla. Se produce en el momento que se libera la tecla.
    • Load (onload): evento que se produce cuando la página se ha terminado de cargar.
    • Mousedown (onmousedown): evento que se produce cuando el usuario pulsa sobre un elemento de la página.
    • Mousemove (onmousemove): evento que se produce cuando el usuario mueve el ratón por la página.
    • Mouseout (onmouseout): evento que se produce cuando el puntero del ratón sale del área ocupada por un elemento de la página.
    • Mouseover (onmouseover): evento que se produce cuando el puntero del ratón entra en el área ocupada por un elemento de la página.
    • Mouseup (onmouseup): evento que se produce cuando el usuario suelta el botón del ratón, que previamente había pulsado.
    • Move (onmove): evento que se produce cuando se mueve la ventana del navegador, o un frame.
    • Resize (onresize): evento que se produce cuando se redimensiona la ventana del navegador, o el frame, en caso de que la página los tenga.
    • Reset (onreset): evento que se produce cuando un usuario hace clic en el botón de reset de un formulario.
    • Select (onselect): evento que se produce cuando un usuario realiza una selección de un elemento de un formulario.
    • Submit (onsubmit): evento que se produce cuando el usuario pulsa el botón para enviar un formulario.
    • Unload (onunload): evento que se produce cuando se avandona la página.
    • DblClick (ondblclick): evento que se produce cuando el usuario hace doble click en un elemento de formulario o un link.
    • Scroll (onscroll): evento que se produce cuando se realiza scroll o desplazamiento.
